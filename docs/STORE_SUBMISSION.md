# Store Submission Guide

How to publish your rebranded app to the Apple App Store and Google Play. This
assumes you've completed the [Rebrand Checklist](REBRAND_CHECKLIST.md) and have a
working `flutter build`.

> You need paid developer accounts: **Apple Developer Program** ($99/yr) and
> **Google Play Console** ($25 one-time).

## 1. Icons & splash

- **Launcher icons** are generated by `flutter_launcher_icons` from
  `assets/icon/icon.png`: `dart run flutter_launcher_icons` (see
  `docs/GETTING_STARTED.md` §6.1). This populates Android mipmaps + adaptive
  icon and the iOS `AppIcon` set.
- **Splash / launch screen**: Android uses the launch theme in
  `android/app/src/main/res`; iOS uses `LaunchScreen.storyboard` in
  `ios/Runner`. Replace the placeholder launch image with your branded one.

## 2. Push notifications — APNs (iOS)

FCM delivers iOS push via APNs, so you must connect them:

1. In the **Apple Developer** portal → Keys → create an **APNs Auth Key** (.p8).
   Note the **Key ID** and your **Team ID**.
2. In the **Firebase Console** → Project Settings → Cloud Messaging → **Apple app
   configuration** → upload the `.p8`, Key ID, and Team ID.
3. Enable the **Push Notifications** and **Background Modes → Remote
   notifications** capabilities in Xcode for the Runner target.
4. Confirm the iOS bundle id matches the Firebase iOS app and
   `GoogleService-Info.plist`.

Android push works once `google-services.json` matches your Firebase project (no
extra cert step).

## 2b. Sign in with Apple — required if you offer Google on iOS

App Store Review **Guideline 4.8** requires that an iOS app offering any
third-party/social login (Google here) **also** offers **Sign in with Apple**.

This product ships with Apple Sign-In **not implemented**, so social sign-in is
**hidden on iOS by default** (`AuthConfig.appleSignInEnabled = false`) — email/
phone login still work, which keeps a default iOS build compliant. To offer
Google on iOS:

1. Implement **Sign in with Apple**: add the `sign_in_with_apple` package, wire
   the Apple button's `onTap` (currently a placeholder), and enable the **Apple**
   provider in Supabase. On the Apple side, create a **Service ID**, a **Sign in
   with Apple key**, and configure return URLs.
2. Set `AuthConfig.appleSignInEnabled = true` (`lib/core/config/auth_config.dart`)
   to re-enable the social section (Apple + Google) on iOS/macOS.

If you don't want social login on iOS, leave it disabled — there's nothing to do.

## 3. Build the release artifacts

```bash
# Android — App Bundle for Play (needs android/key.properties + keystore)
flutter build appbundle --release --obfuscate --split-debug-info=build/symbols

# iOS — archive (run on macOS with Xcode)
flutter build ipa --release --obfuscate --split-debug-info=build/symbols
```

> **Signing**: Android reads `android/key.properties` (gitignored) — generate an
> upload keystore with `keytool` and reference it there. iOS signing is managed
> in Xcode / App Store Connect with your Apple certificates & provisioning.

## 4. Google Play — Data safety

Play requires a **Data safety** form. This app, as shipped, typically collects:

- **Account info**: name, email, phone (auth + profile).
- **Photos / documents**: avatars, shipment photos, KYC documents (private).
- **Location**: approximate/precise if you enable location features.
- **Messages**: chat content (text + voice notes) between users.
- **Device identifiers**: FCM push token.

Declare how each is used, whether it's shared, and that data is encrypted in
transit. Review your actual configuration before submitting — remove categories
you disable.

## 5. Apple — App Privacy ("Nutrition label")

App Store Connect asks the equivalent **App Privacy** questionnaire. Map the same
data categories (Contact Info, User Content, Identifiers, Location if used) and
declare usage/linking/tracking. The app does not include third-party ad/tracking
SDKs by default — declare accordingly.

## 6. Listing assets you'll need

- Screenshots per device class (phone, and tablet if you support it) for both
  stores.
- App icon (stores re-use the launcher icon art; Apple wants a 1024×1024 PNG).
- Short + full description, keywords, category, support URL, privacy-policy URL
  (host `docs/PRIVACY_POLICY.html` — the Firebase legal Hosting site works).

## 7. Pre-submit checklist
- [ ] Bundle/application id matches Firebase apps and store listings.
- [ ] APNs key uploaded to Firebase (iOS push tested on a real device).
- [ ] Signed release build installs and runs on a physical device.
- [ ] Privacy policy + terms hosted and linked.
- [ ] Data safety (Play) + App Privacy (Apple) forms completed accurately.
- [ ] Version + build number bumped in `pubspec.yaml` and `CHANGELOG.md`.
